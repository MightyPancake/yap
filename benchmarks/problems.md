1. Naive recursive Fibonacci — pure recursion / function-call overhead, no allocation.
2. N-Queens backtracking — recursion + heavy data-dependent branching/pruning (distinct from fib's unconditional branching).
3. Quicksort or Mergesort on a large `i32` array — divide-and-conquer recursion combined with in-place array mutation (the one case that's both recursive *and* array-heavy).
4. Longest Common Subsequence (or Levenshtein edit distance) on byte buffers — bottom-up 2D DP table; contrasts with #1 by reusing overlapping subproblems instead of recomputing.
5. Dense N×N matrix multiplication (`f64`) — nested-loop floating point, cache/memory-bandwidth bound, no recursion.
6. Binary tree build → traverse → free — heap-allocation/pointer-chasing heavy; the interesting cross-language angle is manual `malloc`/`free` (yap/C/Rust/Zig/C++) vs GC/ARC (Go/Nim).
7. Character-frequency histogram over a generated byte buffer (256-bucket array) — sequential byte iteration, stand-in for "string processing" since yap has no real string type yet (byte@ only).
8. Hash table insert/lookup throughput, integer-keyed — via a new `modules/hashmap` (see below).
